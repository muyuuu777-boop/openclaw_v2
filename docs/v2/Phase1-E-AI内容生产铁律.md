# Phase1-E · AI 内容生产铁律（Lobster V2）

更新时间：2026-02-27
负责人：AME（main）
适用范围：AI 网红（图/视频）+ AI 微电影 + 系统维护协同

---

## 1) 我们的铁律（正式版）

1. **交付优先，不空转优化**
   - 先产出可交付版本，再做精修。

2. **一条需求，一个落点**
   - 每条需求必须绑定：频道 + 标签 + scope。

3. **先小样，再批量**
   - 任何新风格先 1 张/1 段验证，确认后再批量渲染。

4. **提示词分层，不混写**
   - 创意层（风格/叙事）+ 技术层（参数/镜头）+ 平台层（工具兼容）分离维护。

5. **风格资产化**
   - 可复用风格必须沉淀模板（Prompt 模板 + 参数卡 + 负面词）。

6. **双层记忆强制落库（借鉴并本地化）**
   - 每次踩坑/经验必须写入 LanceDB 两条：
     - 技术层（fact）：症状/根因/修复/预防
     - 原则层（decision）：触发条件/决策规则/执行动作
   - 写入后必须检索验证命中，未命中不算完成。

7. **记录可追溯，口头不算完成**
   - 完成定义 = 产物链接 + 回执 + 关键参数 + 负责人 + 时间。

8. **失败必须复盘**
   - 失败要写“失败原因 + 下一次怎么避免”，禁止只说“重跑了”。

9. **实验隔离，生产稳态**
   - 新工具/新模型先在 LAB 验证，确认后再进生产链路。

10. **客户域隔离，方法可迁移**
    - KS / BR / FILM 数据与上下文隔离；可迁移的是抽象方法，不是原始素材。

11. **系统改动四问（审计铁律）**
    - 改了什么 / 做了什么 / 为什么改 / 为什么回滚。

12. **默认可回滚**
    - 任何更新都要有回退路径与验证步骤。

---

## 2) 与 Blogger 技术铁律的映射关系

- 原则“失败即记忆” → 我们保留，并增强为“**双层记忆 + 检索验收**”。
- 原则“重试前先 recall” → 我们保留，避免盲目重跑耗时耗 token。
- 原则“避免写错代码库” → 我们扩展为“避免写错项目域/客户域”。
- 原则“插件变更后清缓存” → 保留为系统维护硬规则（适用于 plugins/*.ts）。

## 3) Bot / Agent 的铁律分配（执行归属）

### AME（main）
- 负责铁律 10/11/12（域隔离、审计、回滚）
- 审批全局原则是否升格到 `global`

### Dispatch / Managers
- 负责铁律 1/2/3（交付优先、任务落点、小样先行）
- 拆单时强制标签与 scope 对齐

### Prompt / Image / Video / Render
- 负责铁律 4/5/7（分层提示词、资产化、可追溯）

### QC / Reporter
- 负责铁律 8（失败复盘）与交付审计闭环

### System-Ops / System-Lab
- 负责铁律 6/9（双层记忆落库、实验隔离）

### 原则层（decision）
`Decision principle ([tag]): [规则]. Trigger: [触发条件]. Action: [执行动作]. Owner: [agent].`

## 5) 生效说明

- 本铁律用于内容生产与系统治理，不用于替代具体 SOP。
